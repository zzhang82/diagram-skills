# Reference Gap Spec

## Objective
Bring presentation-mode outputs closer to the polished reference concept.

## Gap summary
The reference concept is calmer, more editorial, and more selective. The renderer output was more system-led, denser, and flatter.

## Differences to close
- Reduce visible detail by roughly 20-30%
- Prefer 4 major zones over many equally weighted blocks
- Use a calmer palette: one primary accent plus restrained secondary semantics
- Prefer warm off-white or subtle tinted backgrounds over flat gray dashboards
- Tighten type hierarchy: stronger title, quieter subtitle, less uniform text sizing
- Make cards lighter and less mechanical
- Integrate explainer panels as support, not as a competing UI column
- Ensure content fits nodes predictably; text should drive card width and line count

## Rendering rules
- Default to presentation mode, not editor chrome
- Use whitespace aggressively between major sections
- Keep explainer cards to 3-5 bullets or short note blocks
- Use multiline labels for hero nodes when it improves balance
- Make connector styling lighter than card styling

## Scope
### In scope
- calmer palettes
- editorial spacing
- compact explainer cards
- multiline title/node labels
- footer callout banner

### Out of scope
- full 3D rendering
- fake dashboard chrome by default
- rainbow category coloring
- overly detailed right rails
